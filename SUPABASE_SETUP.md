## 📋 Next Steps

### 1. Get Your Service Role Key (Backend Authentication)
Since the backend needs to authenticate as a service, you'll need your **Service Role Key** (not the public key).

1. Go to **Supabase Dashboard** → Your Project
2. Click **Settings** (gear icon) in the left sidebar
3. Click **API** tab
4. Copy the **Service Role Key** (labeled "secret")
   - This is different from the Anon Public Key you already have

### 2. Create Database Tables in Supabase

In your Supabase dashboard, go to **SQL Editor** and run these commands:

```sql
-- Create users table with UUID (compatible with Supabase auth)
CREATE TABLE users (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  username TEXT UNIQUE NOT NULL,
  email TEXT UNIQUE NOT NULL,
  password_hash TEXT NOT NULL,
  is_admin BOOLEAN DEFAULT false,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP
);

-- Create orders table
CREATE TABLE orders (
  id BIGINT PRIMARY KEY GENERATED BY DEFAULT AS IDENTITY,
  user_id UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  game_title TEXT,
  package_label TEXT,
  uid TEXT,
  server_id TEXT,
  quantity INTEGER DEFAULT 1,
  price REAL DEFAULT 0,
  total REAL DEFAULT 0,
  esewa_code TEXT,
  status TEXT DEFAULT 'Pending',
  timestamp TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP
);

-- Create indexes for better performance
CREATE INDEX idx_orders_user_id ON orders(user_id);
CREATE INDEX idx_users_username ON users(username);
CREATE INDEX idx_users_email ON users(email);
```

### 4. Enable Row-Level Security (Optional but Recommended)

In Supabase SQL Editor, run:
```sql
-- Enable RLS
ALTER TABLE users ENABLE ROW LEVEL SECURITY;
ALTER TABLE orders ENABLE ROW LEVEL SECURITY;

-- Add policies (Fixed for UUID type)
CREATE POLICY "Users can read their own data"
  ON users FOR SELECT
  USING (auth.uid() = id OR is_admin = true);

CREATE POLICY "Users can read their own orders"
  ON orders FOR SELECT
  USING (auth.uid() = user_id OR 
         (SELECT is_admin FROM users WHERE id = auth.uid()));
```

## 🔍 Testing

Once configured, test the connection:

1. **From Frontend**: Open browser console and run:
```javascript
const { data, error } = await supabase.from('users').select('*').limit(1);
console.log(data, error);
```

2. **From Backend**: Run test script
```bash
python -c "
from app_supabase import supabase
try:
    users = supabase.table('users').select('*').limit(1).execute()
    print('✅ Connected:', users.data)
except Exception as e:
    print('❌ Error:', e)
"
```

## 🆘 Troubleshooting

**Issue**: "Table does not exist" error
- **Solution**: Run the SQL commands in Supabase SQL Editor

**Issue**: "Invalid JWT" error
- **Solution**: Verify your Service Role Key is correct (not public key)

**Issue**: Frontend can't connect
- **Solution**: Check if Supabase JS library is loaded in browser DevTools → Network tab
